PART 2 — STEP BY STEP SETUP ON YOUR MACHINE
========================================================

STEP 1: Install Node.js (if you don't have it)
────────────────────────────────────────────────
Go to: https://nodejs.org
Download the LTS version (20.x or higher).

Check it works:
  node --version    # should show v20.x.x
  npm --version     # should show 10.x.x

STEP 2: Copy the project to your machine
─────────────────────────────────────────
Option A — From the download:
  Unzip the project folder.
  Put it somewhere like:
  ~/projects/react-native-accessibility-scanner

Option B — Create a GitHub repo first (recommended):
  See PART 3 below, then clone it.

STEP 3: Open the folder in your terminal
─────────────────────────────────────────
  cd ~/projects/react-native-accessibility-scanner

STEP 4: Install dependencies
──────────────────────────────
  npm install

This installs:
  - @babel/parser
  - @babel/traverse
  - @babel/types
  - chalk
  - commander
  - cosmiconfig
  - glob
  - vitest (for tests)
  - typescript

STEP 5: Run the tests
──────────────────────
  npm test

You should see:
  ✓ tests/unit/rules.test.ts (24 tests)
  ✓ tests/unit/ast-utils.test.ts (19 tests)
  ✓ tests/unit/scanner.test.ts (11 tests)
  ✓ tests/unit/reporters.test.ts (9 tests)
  Tests 63 passed (63)

STEP 6: Build the project
──────────────────────────
  npm run build

This compiles TypeScript → JavaScript in the dist/ folder.

STEP 7: Test the CLI locally
──────────────────────────────
  node dist/cli/index.js ./src --score

You should see the colored accessibility report.

========================================================
PART 3 — CREATE YOUR GITHUB REPOSITORY
========================================================

STEP 1: Create account
  Go to: https://github.com
  Sign up or sign in.

STEP 2: Create new repository
  Click the "+" button → "New repository"
  Name: react-native-accessibility-scanner
  Description: Accessibility auditing for React Native apps.
  Set to Public
  Do NOT check "Add README" (we already have one)
  Click "Create repository"

STEP 3: Push your code to GitHub
  In your project folder, run:

  git init
  git add .
  git commit -m "feat: initial release v0.1.0"
  git branch -M main
  git remote add origin https://github.com/YOUR_USERNAME/react-native-accessibility-scanner.git
  git push -u origin main

  Replace YOUR_USERNAME with your actual GitHub username.

STEP 4: Add the GitHub Action
  Copy the workflow file:

  mkdir -p .github/workflows
  cp github-action/accessibility.yml .github/workflows/accessibility.yml
  git add .github/
  git commit -m "ci: add accessibility scan workflow"
  git push

  Now every push and PR will automatically run the scanner.

STEP 5: Add topics to your repo (helps people find it)
  On GitHub, go to your repo page.
  Click the gear icon next to "About".
  Add topics:
    react-native
    accessibility
    a11y
    eslint
    cli
    typescript
    linting

PART 4 — PUBLISH TO NPM
========================================================

STEP 1: Create an npm account
  Go to: https://www.npmjs.com
  Click "Sign Up"
  Choose a username (this appears in your package URL)
  Verify your email address

STEP 2: Log into npm from your terminal
  npm login

  It will ask for:
  - Username
  - Password
  - Email
  - One-time password (from your email or 2FA app)

STEP 3: Update package.json with YOUR details
  Open package.json and change these lines:

  "author": "Your Name <your@email.com>",
  "repository": {
    "url": "https://github.com/YOUR_USERNAME/react-native-accessibility-scanner"
  },
  "homepage": "https://github.com/YOUR_USERNAME/react-native-accessibility-scanner#readme",
  "bugs": {
    "url": "https://github.com/YOUR_USERNAME/react-native-accessibility-scanner/issues"
  }

STEP 4: Check the package name is available
  npm search react-native-accessibility-scanner

  If it shows "No matches found" → the name is yours.
  If taken, try:
    @YOUR_USERNAME/react-native-accessibility-scanner
  (This is a "scoped" package)

STEP 5: Do a dry run first (see what will be published)
  npm publish --dry-run

  This shows exactly which files will be uploaded.
  You should see:

PART 6 — VERSIONING WORKFLOW (how to release updates)
========================================================

When you FIX a bug (0.1.0 → 0.1.1):
  1. Fix the bug
  2. npm test  (make sure tests pass)
  3. npm version patch  (auto-bumps version)
  4. npm run build
  5. npm publish
  6. git push && git push --tags

When you ADD a feature (0.1.0 → 0.2.0):
  1. Add the feature
  2. Add tests for it
  3. npm test
  4. npm version minor
  5. npm run build
  6. npm publish
  7. git push && git push --tags

When you make a BREAKING CHANGE (1.x → 2.0.0):
  1. npm version major
  2. npm run build
  3. npm publish
  4. git push && git push --tags

RELEASE CHECKLIST (every time):
  □ npm test passes (63/63)
  □ npm run build succeeds
  □ README updated if needed
  □ package.json version bumped
  □ Committed to git
  □ Tagged in git (npm version does this automatically)
  □ npm publish
  □ Test: npx react-native-accessibility-scan --version
